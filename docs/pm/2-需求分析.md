## 第一部分：需求分析汇总

### 一、业务需求

> **核心问题**: 产品要帮业务实现什么目标？为什么要做？值不值得做？

#### 1.1 商业目标

| 目标 | 指标 | 当前值 | 目标值 | 期限 |
|------|------|--------|--------|------|
| 验证「Web BLE→激光笔顺投影」可行 | 原型跑通核心闭环 | 0 | 1 | 2026-08 |
| 验证市场需求 | 演示视频互动量 | 0 | 点赞 1000+ | 2026-09 |
| 建立开源社区 | GitHub Stars | 0 | 200 | 2026-10 |
| 首次套件销售 | 付费订单 | 0 | 30 单 | 2026-12 |

#### 1.2 价值衡量维度

| 维度 | 预期效果 | 量化方式 |
|------|---------|---------|
| 用户情感价值 | 令人惊喜，愿意分享 | 用户自传播率 ≥ 30% |
| 社区价值 | 开源贡献飞轮 | 3 个月内 ≥ 5 个外部 PR |
| 分发效率 | URL 打开即可用 | 打开→首次投影 ≤ 30 秒 |

#### 1.3 干系人分析

| 干系人 | 角色 | 诉求 | 影响程度 | 参与阶段 |
|--------|------|------|---------|---------|
| 创作者 (Yangy) | PM + 全栈 | 给女朋友惊喜，开源建社区 | 高 | 全过程 |
| 女朋友 | 终端用户 V0 | 墙上看到字投影出来，拍照发朋友圈 | 高 | Alpha |
| 科技 DIY 爱好者 | 核心用户 | 有趣的 ESP32 项目，焊接不难 | 中 | Beta→GA |
| 情侣礼物购买者 | 付费用户 | 送对象，操作傻瓜，包装好看 | 中 | GA 后 |
| 家长 (亲子教育) | 扩展用户 | 投影汉字笔顺，孩子觉得有趣 | 低 | V1.1+ |

#### 1.4 关键商业假设

| 假设 | 如果错误 | 验证方法 |
|------|---------|---------|
| Web Bluetooth 连接成功率可接受 | 用户连不上就流失 | 5 台不同品牌手机实测 |
| 激光投影效果足够惊艳 | 看起来不够魔法 | 买振镜+激光最简验证 |
| 换电池体验优于充电 | 用户觉得买电池麻烦 | A/B 对比 |
| iOS 用户能用第三方浏览器凑合 | iOS 占比高且不愿装第三方浏览器 | 统计用户平台分布，iOS > 40% 则补做 APP |

> **Web Bluetooth 关键限制**: 需要 HTTPS (GitHub Pages 默认满足)；iOS Safari 不支持 (需引导用 Bluefy 或等待 WebKit 支持)；部分旧 Chrome 需要 flag 开启。

### 四、数据需求

> **核心问题**: 系统需要存储和管理哪些数据？

#### 4.1 领域模型

```
┌──────────┐       ┌──────────┐       ┌──────────┐
│  Project │──1:N──│  Stroke  │──1:N──│  Point   │
│ • id      │       │ • index   │       │ • x,y,t,p│
│ • name    │       │ • color   │       └──────────┘
│ • created │       │ • weight  │
└──────────┘       └──────────┘
```

#### 4.2 数据实体清单

| 实体 | 存储位置 | 预估量 | 保密级别 |
|------|---------|--------|---------|
| Project | IndexedDB (浏览器) | ~500 条 | 内部 |
| Stroke | IndexedDB | ~5000 条 | 内部 |
| Point | IndexedDB | ~50 万条 | 内部 |
| 设备配置 | ESP32 NVS | 1 条 | 内部 |
| PWA 缓存 | Service Worker Cache | ~5MB | 公开 |

#### 4.3 数据流转

```
[用户触摸] ──[60Hz]──> [Canvas2D + Atrament.js]
                              │
                [笔画序列 JSON / 二进制]
                              │
            ┌─────────────────┴──────────┐
            │  Web Bluetooth API           │
            │  navigator.bluetooth         │
            │  requestDevice() → GATT      │
            │  writeCharacteristicValue()  │
            │  分包 (MTU 180+ B)           │
            └─────────────────┬──────────┘
                              │
            ┌─────────────────┴──────────┐
            │  ESP32 BLE 回调              │
            │  → 环形缓冲                  │
            │  → 组包 + CRC8 校验          │
            │  → 解析为 Stroke[]           │
            └─────────────────┬──────────┘
                              │
            ┌─────────────────┴──────────┐
            │  矢量重建引擎                │
            │  → DAC 电压映射              │
            │  → 双缓冲 DMA 输出           │
            │  → 振镜扫描 + 激光调制       │
            └────────────────────────────┘
```

#### 4.4 BLE 协议帧

```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────────┬──────┐
│ 0xAA │ 0x55 │ Type │ Stk# │ Total│ Seq# │ Payload  │ CRC8 │
│ 2B   │      │ 1B   │ 1B   │ 1B   │ 1B   │ N 字节    │ 1B   │
└──────┴──────┴──────┴──────┴──────┴──────┴──────────┴──────┘

Type: 0x01=数据 0x02=投影指令 0x03=停止 0x04=配置 0x05=ACK
```

#### 4.5 数据生命周期

| 阶段 | 存储 | 处理 |
|------|------|------|
| 书写中 | 浏览器 JS heap | 实时 |
| 已保存 | IndexedDB | 浏览+回放 |
| 冷数据 (>30 天) | 自动清理 (LRU) | 删除 |
| ESP32 缓存 | 投影完成后清除 | 不持久化 |

#### 4.6 数据埋点 (可选，本地匿名)

| 事件 | 触发时机 | 用途 |
|------|---------|------|
| page_load | 网页打开 | PV |
| ble_connect | 连接成功 | 设备统计 |
| writing_start | 开始书写 | 功能使用率 |
| projection_start | 点击投影 | 功能使用率 |
| projection_complete | 投影完成 | 完播率 |
| ble_error | 连接异常 | 质量监控 |

### 五、非功能需求

> **核心问题**: 功能要做到什么质量标准？

#### 5.1 NFR 总览

| 类别 | 核心指标 | 目标值 |
|------|---------|--------|
| **性能** | 书写/传输/投影延迟 | 见 5.2 |
| **安全** | 激光安全 + Web 安全 + 电气安全 | 见 5.3 |
| **可用性** | 打开即用、换电池简单 | 见 5.4 |
| **兼容性** | 浏览器、手机型号 | 见 5.5 |

#### 5.2 性能

| 指标 | 目标值 | 测量方法 |
|------|--------|---------|
| Canvas 书写延迟 | ≤ 30ms | rAF 帧时间 |
| Web Bluetooth 连接耗时 | ≤ 3s | navigator.bluetooth.requestDevice 计时 |
| BLE 单包延迟 | ≤ 50ms | 发送→ACK 时间 |
| 投影启动延迟 | ≤ 200ms | 点投影→激光扫描 |
| 振镜扫描速率 | ≥ 30kpps | ESP32 定时器 |
| Web App 首屏加载 | ≤ 2s (4G) | Lighthouse |
| PWA 离线启动 | ≤ 1s (已缓存) | 离线测量 |

#### 5.3 安全

| 类别 | 要求 | 优先级 |
|------|------|--------|
| **HTTPS** | Web Bluetooth 仅支持安全上下文 | P0 (GitHub Pages 默认) |
| **激光等级** | Class 2 (≤1mW) | P0 |
| **静止关断** | 振镜 5s 无动作→关激光 | P0 |
| **晃动关断** | MPU6050 检测→立即关 | P0 |
| **物理开关** | 机械开关关激光电源 | P0 |
| **安全标签** | 外壳贴警告标签 | P0 |
| **Web 安全** | CSP: default-src 'self' | P0 |
| **电池反接** | 二极管/PMOS 保护 | P0 |
| **眼球安全** | 网页首页弹窗「激光禁止直射眼睛」 | P0 |

#### 5.4 可用性与可靠性

| 指标 | 目标值 |
|------|--------|
| Web Bluetooth 连接成功率 | ≥ 90% (Android Chrome) |
| BLE 断线重连 | ≤ 30s，自动 |
| 电池续航 (4×AA 碱性) | ≥ 30 分钟连续投影 |
| 电池续航 (4×AA eneloop) | ≥ 45 分钟 |
| 换电池耗时 | ≤ 30 秒 |
| 低电指示 | < 4.0V LED 闪烁 |

#### 5.5 兼容性

| 平台/组件 | 最低版本 |
|------|---------|
| Chrome (Android) | ≥ 56 (Web Bluetooth) |
| Chrome (Desktop) | ≥ 56 |
| Edge | ≥ 79 |
| Samsung Internet | ≥ 6.0 |
| Safari (iOS) | ❌ 不支持 Web Bluetooth |
| iOS 替代方案 | Bluefy 浏览器 / WebBLE |
| ESP32 | S3 / C3 / C6 ≥ 4MB Flash |
| 振镜 | 20kpps 模拟 ±5V |

#### 5.6 硬件安全

| 项 | 要求 |
|------|------|
| 激光出光口 | 保护罩，防手指伸入 |
| 外壳 | 3D 打印，无尖锐边角 |
| 电池仓 | 防反接 |
| 散热 | 自然通风 (功耗 < 10W) |
| 尺寸 | 目标 < 120×80×50mm |

4. **传播点**: 女友/朋友看到投影的瞬间

## 第四部分：用户故事与验收标准

> P0 = MVP, P1 = V1, P2 = 未来

### Epic 1: 核心书写投影闭环

#### US-01 [P0]: Web App 手写画布

**用户故事**: 作为一个用户，我希望在 Chrome 浏览器打开网页后就能用手指书写，无需安装任何 APP。

**验收标准**:

| AC# | Given | When | Then |
|-----|-------|------|------|
| AC1 | 手机 Chrome 打开 inklight.app | Canvas 加载完成 | 看到书写区域，可直接用手指书写 |
| AC2 | 书写中 | 手指滑动 | 笔迹实时渲染，延迟 ≤ 30ms |
| AC3 | 书写中 | 手掌误触 | 系统忽略大面积触摸 |
| AC4 | 首次访问 | 页面加载 | Service Worker 注册，可后续离线使用 |
| AC5 | 用户点"添加到桌面" | PWA manifest 生效 | 桌面出现 InkLight 图标 |

#### US-02 [P0]: Web Bluetooth 配对与数据发送

**用户故事**: 作为一个用户，我希望点击"连接设备"后自动扫描并配对投影盒，无需输入配对码。

**验收标准**:

| AC# | Given | When | Then |
|-----|-------|------|------|
| AC1 | 投影盒已上电 | 点击"连接设备" | 弹出 Chrome 设备选择框，显示 InkLight |
| AC2 | 选择设备 | 配对完成 | 显示"已连接"，画布可投影 |
| AC3 | 已连接，完成书写 | 点击"投影" | BLE 分包发送，MTU 自动协商 |
| AC4 | BLE 断连 | 30s 内 | 自动重连并提示 |
| AC5 | iOS Safari 打开 | 页面检测到不支持 | 显示引导：「请用 Chrome 打开，或安装 Bluefy」 |

**异常场景**:
- 用户取消设备选择 → 保持扫描状态，提示"请选择设备"
- 设备不在范围内 → 提示"未发现设备，请确认电源已开"
- MTU 协商失败 → 降级 20 字节/包，提示"兼容模式"

#### US-03 [P0]: 激光振镜投影回放

**用户故事**: 作为一个用户，我希望笔画数据发送后，激光在墙上按笔顺和原始速度一笔一画还原。

**验收标准**:

| AC# | Given | When | Then |
|-----|-------|------|------|
| AC1 | ESP32 收到完整数据 | 开始投影 | 激光逐笔扫描，先第 1 笔再第 2 笔... |
| AC2 | 投影中 | 当前笔画扫描 | 速度与原始书写一致 |
| AC3 | 全部笔画完成 | 收笔 | 激光保持 2s 后关闭 |
| AC4 | 用户点"停止" | 发送停止指令 | ESP32 立即关激光 |

#### US-08 [P0]: 激光安全机制
