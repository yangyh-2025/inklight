## 第一章: 需求背景与目标

### 1.1 需求背景

华为「随心画」让手写投影成为社交平台爆款内容。微信小程序方案虽可行，但面临 BLE 权限审核风险（可能卡几周甚至被拒）。**Web Bluetooth API** 是更优雅的选择——Chrome 56+ 即原生支持，打开网页就能配对设备，无需审核、无需安装、即时更新。

唯一代价是 iOS Safari 不支持，但可通过引导用户安装 Bluefy 等第三方浏览器解决，或等待 WebKit 逐步支持（已列入路线图）。

### 1.2 业务目标

| 目标 | 指标 | 当前值 | 目标值 | 期限 |
|------|------|--------|--------|------|
| 原型验证 | 核心闭环跑通 | 0 | 1 | 2026-08 |
| 社区基础 | GitHub Stars | 0 | 200 | 2026-10 |
| 首次销售 | 套件/成品订单 | 0 | 30 | 2026-12 |

### 1.3 用户价值

- **零摩擦**: 一个 URL 打开即用——微信/短信/二维码都能分享
- **零审核**: 功能更新即时生效，不需要等微信审核
- **零安装**: PWA 安装到桌面就是伪原生体验，不占存储空间
- **真正便携**: 口袋大小的投影盒 + 4 节 AA 电池

## 第二章: 范围定义

### 2.1 In Scope

- **Web App (单页应用)**: Canvas 手写画布 (Atrament.js) + 笔画录制 + Web Bluetooth API 数据发送 + PWA (Service Worker + Manifest)
- **ESP32 固件**: BLE 外设 (GATT Server) + 接收组包 + 矢量插值 + 双缓冲 DMA DAC → 振镜
- **投影盒硬件**: ESP32-S3 + MCP4822 DAC + 20kpps 振镜 + 红绿双色激光 + 4AA 电池盒 + 3D 打印外壳
- **BLE 自定义协议**: 帧格式 (MTU 180+ 字节)，丢包重传
- **激光安全**: 静止关断 + 晃动关断 + 物理钥匙开关
- **托管**: GitHub Pages (HTTPS 免费，Web Bluetooth 必需安全上下文)
- **PWA**: 可安装到桌面、离线使用 (已保存的作品)、Service Worker 缓存

### 2.2 Out of Scope

- **不做 iOS Safari 适配** — Safari 不支持 Web Bluetooth (V1.0 引导使用 Bluefy)
- **不做独立 iOS/Android APP** — Web App 足够
- **不做微信小程序** — 避免审核
- **不做内置锂电** — 只用 AA 换电
- **不做 WiFi/云端同步** — V1.0 纯本地
- **不做 RGB 彩色** — V1.0 双色
- **不做 AI 书法** — V2.0

### 4.2 核心流程

#### 主流程: 网页打开 → 配对 → 写字 → 投影

**正常流程:**

1. 用户用 Chrome 打开 `https://inklight.app` (或 GitHub Pages URL)
2. 页面加载，显示手写画布，顶部显示"未连接"
3. 投影盒装入 4 节 AA 电池，打开电源开关
4. 用户点击"连接设备"
5. `navigator.bluetooth.requestDevice()` 弹出 Chrome 设备选择器
6. 用户选择 "InkLight-XXXX"
7. 配对成功，顶部显示"已连接 ✓"，画布进入可书写状态
8. 用户在 Canvas 上手指书写
9. Atrament.js 实时录制笔画数据
10. 用户写完，点击"▶ 投影"
11. Web Bluetooth API 按 BLE GATT Write 分包发送
12. ESP32 接收→组包→校验→矢量插值→DAC→振镜扫描
13. 墙面激光逐笔还原书写过程
14. 回放完毕，2s 后自动关激光
15. 网页显示"投影完成 ✓"

**异常流程:**

- 浏览器不支持 Web Bluetooth → 显示「当前浏览器不支持，请用 Chrome 打开」
- iOS Safari → 显示「iOS Safari 暂不支持，请安装 Bluefy 浏览器 (免费)」
- 扫描不到设备 → 「未发现设备：①电源开了吗 ②距离 10m 内吗」
- MTU 协商失败 → 降级 20 字节自动适配
- BLE 中途断连 → 30s 自动重连，ESP32 缓存未完成数据
- 投影中用户点停止 → 立即发送停止指令，关激光

**边界条件:**

- 超长笔画 (100+ 点): 分包 > 5，重传机制保障
- 连续快速点投影: 防抖 2s
- 页面后台/锁屏: 保持 BLE 连接 (Navigator Wake Lock API)

### 4.5 Web Bluetooth 配对流程

```javascript
// 核心配对代码
const SERVICE_UUID = '0000ffe0-0000-1000-8000-00805f9b34fb';
const TX_CHAR_UUID = '0000ffe1-0000-1000-8000-00805f9b34fb'; // Notify
const RX_CHAR_UUID = '0000ffe2-0000-1000-8000-00805f9b34fb'; // Write

async function connect() {
  const device = await navigator.bluetooth.requestDevice({
    filters: [{ namePrefix: 'InkLight-' }],
    optionalServices: [SERVICE_UUID]
  });
  const server = await device.gatt.connect();
  const service = await server.getPrimaryService(SERVICE_UUID);
  const txChar = await service.getCharacteristic(TX_CHAR_UUID);
  const rxChar = await service.getCharacteristic(RX_CHAR_UUID);
  // txChar 用于接收 ESP32 状态/确认
  // rxChar 用于发送笔画数据
  await txChar.startNotifications();
  txChar.addEventListener('characteristicvaluechanged', handleNotification);
  return { device, server, rxChar };
}
```

### 4.6 ESP32 BLE GATT Server

```c
// 关键 GATT 服务定义
#define SERVICE_UUID          "0000ffe0-0000-1000-8000-00805f9b34fb"
#define CHARACTERISTIC_TX_UUID "0000ffe1-0000-1000-8000-00805f9b34fb"  // Notify
#define CHARACTERISTIC_RX_UUID "0000ffe2-0000-1000-8000-00805f9b34fb"  // Write

// RX 回调: 接收手机发来的笔画数据包
void onWrite(BLECharacteristic *pCharacteristic) {
    uint8_t *data = pCharacteristic->getData();
    size_t len = pCharacteristic->getValue().length();
    packetBuffer.push(data, len);
    if (packetComplete()) {
        parseAndEnqueueStroke(packetBuffer.assemble());
    }
}
```

### 4.7 ESP32 固件架构

```
Core 0: BLE 回调
  ├── onWrite() → 环形缓冲 (ringbuf)
  ├── 组包校验 (帧头 0xAA55 + CRC8)
  └── 解析为 Stroke[]

Core 1: 投影引擎
  ├── 从 Stroke[] 取下一采样点
  ├── 坐标→DAC 电压 (12-bit → 运放 → ±5V)
  ├── SPI DMA 双缓冲输出
  ├── 定时器触发 (30kpps)
  └── 笔画切换时控制激光颜色

安全任务 (FreeRTOS, 优先级最高)
  ├── MPU6050 读数
  ├── |accel| > 阈值 → GPIO 紧急关断激光
  ├── 振镜空闲 > 5s → GPIO 关断激光
  └── 物理开关状态检测
```

### 4.9 硬件接线

```
ESP32-S3
  ├── GPIO18 (SCK)  ──── MCP4822 SCK
  ├── GPIO23 (MOSI) ──── MCP4822 SDI
  ├── GPIO5  (CS)   ──── MCP4822 CS
  ├── GPIO25         ──── 红色激光驱动 (PWM/GPIO)
  ├── GPIO26         ──── 绿色激光驱动 (PWM/GPIO)
  ├── GPIO32         ──── 安全继电器/PMOS (激光总电源)
  ├── GPIO33         ──── 低电指示灯
  ├── I2C (21/22)    ──── MPU6050 加速度计
  └── 3.3V ← 降压模块 (12V→3.3V)

MCP4822 → 运放 TL084 → 振镜 (±5V)
  4×AA (6V) → MT3608 升压 (12V) → 振镜/激光/降压
```

## 第五章: 数据需求

### 5.1 领域模型 (Web App 端, IndexedDB)

```
Project (id, name, canvas_w, canvas_h, stroke_count, thumbnail, created_at)
  │ 1:N
Stroke (id, project_id, stroke_index, color, weight, mode, start_ms, end_ms)
  │ 1:N
Point (stroke_id, seq, x, y, t_offset_ms, pressure)
```

### 5.2 BLE 传输帧格式

```
┌──────┬──────┬──────┬──────┬──────┬──────┬──────────┬──────┐
│ 0xAA │ 0x55 │ Type │ Stk# │ Total│ Seq# │ Payload  │ CRC8 │
│ 2B   │      │ 1B   │ 1B   │ 1B   │ 1B   │ N 字节    │ 1B   │
└──────┴──────┴──────┴──────┴──────┴──────┴──────────┴──────┘

Type: 0x01=笔画数据 0x02=开始投影 0x03=停止 0x04=设参 0x05=ACK

Payload (Type=0x01):
  颜色(1B) + 粗细(1B) + 坐标点序列
  每点: x(uint16_le, 2B) + y(uint16_le, 2B) + dt(uint8, 10ms单位, 1B)
  = 每点 5 字节

MTU=185 → 有效载荷 ~176B → 每包可传约 35 个坐标点
MTU=20  → 有效载荷 ~11B  → 每包可传约 2 个坐标点 (降级模式)
```

### 5.3 数据生命周期

| 阶段 | 存储 | 处理 |
|------|------|------|
| 书写中 | 浏览器 JS heap | 实时 |
| 已保存 | IndexedDB | 浏览+回放 |
| >30 天 | LRU 自动清理 | 删除 |
| ESP32 缓存 | 投影完清除 | 不持久化 |
