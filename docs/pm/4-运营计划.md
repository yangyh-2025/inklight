## 第一部分：风险评估

### 1. 风险矩阵

```
     影响
      ↑
 高   │  激光安全事件    │  核心体验不达预期
      │  iOS用户流失     │
      │                 │
 中   │  振镜质量差      │  Web BT 兼容性
      │                 │  社区冷启动
      │                 │
 低   │  外壳打印失败    │  电池仓尺寸偏差
      │                 │
      └─────────────────┴──────────────→ 概率
         低                 高
```

### 2. 市场风险

| 风险 | 概率 | 影响 | 缓解策略 | 触发信号 |
|------|------|------|---------|---------|
| iOS 用户占比高无法使用 | 中 | 高 | 统计 Web App UA 分布；iOS > 40% 则补做独立 APP；引导 Bluefy | iOS 访问 > 40% |
| 激光概念吓退普通用户 | 中 | 中 | 强调 Class 2 (和激光笔一样)；外壳可爱化设计 | 用户反馈不敢用 |
| 送礼需求季节性 | 中 | 中 | 拓展亲子教育/市集场景 | 月销量波动 > 80% |

### 3. 技术风险

| 风险 | 概率 | 影响 | 缓解策略 | 触发信号 |
|------|------|------|---------|---------|
| **核心体验不达预期** | 中 | 高 | 最简验证——先买振镜+激光不买外壳，跑通 BLE→墙面 | 朋友说"就这？" |
| **Web Bluetooth 兼容性** | 中 | 中 | 测试 5+ 台不同品牌 Android 机 + Chrome 版本；提供兼容性检测页 | 某机型连不上 |
| MTU 协商失败 | 中 | 低 | 自动降级 20 字节/包，网页提示"兼容模式" | - |
| 振镜质量不稳定 | 中 | 中 | 多供应商测试；BOM 标注可信店铺 | 投影抖动/漂移 |
| PWA 缓存策略 bug | 低 | 中 | 测试离线场景 (飞行模式)；Service Worker 版本管理 | 离线白屏 |

### 4. 运营风险

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|---------|
| GitHub 冷启动 | 中 | 中 | 提前做视频预热；"华为同款 ¥249"标题钩子 |
| 文档不清晰 | 高 | 中 | 录 3 分钟视频；找非技术朋友试装 |
| 套件库存积压 | 低 | 低 | 按需采购，不预置库存 |

### 5. 法律/合规风险

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|---------|
| 激光安全事故 | 低 | 极高 | Class 2 + 三重关断 + 安全标签 + 网页警告 |
| 商标纠纷 | 低 | 低 | 不用华为商标，项目名独立 |

### 6. 最重要的 3 个风险

1. **核心体验不达预期** — 激光不够亮/不够丝滑 = 整个项目没价值。必须最简验证后再投入
2. **激光安全** — 一次事故毁掉一切。三重冗余保护不是可选项
3. **iOS 用户无法使用** — Web Bluetooth 不兼容 Safari。先看用户分布再决定是否补 APP

## 第二部分：决策记录

### D-001: 激光振镜而非 DLP 投影

| 字段 | 内容 |
|------|------|
| **日期** | 2026-06-27 |
| **决策** | ESP32 + 激光振镜替代树莓派 + DLP2000 |
| **原因** | 手写=矢量线条，振镜天然匹配。成本从 ¥1120→¥246 |
| **可逆性** | 可逆 (抽象 Projector 接口) |

### D-002: Web Bluetooth API 而非微信小程序

| 字段 | 内容 |
|------|------|
| **日期** | 2026-06-27 |
| **决策者** | Yangy |
| **决策** | 使用 Web Bluetooth API + PWA，放弃微信小程序方案 |
| **背景** | 小程序需要 BLE 权限审核，可能卡几周甚至被拒。Web Bluetooth 无审核、无安装、Chrome 56+ 原生支持。更新即时生效 |
| **备选方案** | 1. 微信小程序 — 需要审核 2. React Native APP — 开发成本高 |
| **代价** | iOS Safari 不支持，需引导用户安装 Bluefy 等第三方浏览器 |
| **可逆性** | 可逆 — 代码是 Web 前端，随时可包装为小程序或 APP |
| **重评估** | iOS 用户占比 > 40% 时考虑补做独立 APP |

### D-003: AA 电池而非锂电

| 字段 | 内容 |
|------|------|
| **日期** | 2026-06-27 |
| **决策** | 4 节 AA 可换电池，不支持充电 |
| **原因** | 低频使用场景，锂电闲置伤电池；换电池体验"没电秒换" |
| **可逆性** | 可逆 — 电池盒模块化，可换 18650 座 |

### D-004: GitHub Pages 托管

| 字段 | 内容 |
|------|------|
| **日期** | 2026-06-27 |
| **决策** | Web App 托管于 GitHub Pages |
| **背景** | 免费 HTTPS (Web BT 必需)、自动部署 (git push)、全球 CDN |
| **影响** | 仅静态文件 (SPA)，适合单页应用；有带宽限制但够用 |

### D-005: MIT 开源

| 字段 | 内容 |
|------|------|
| **日期** | 2026-06-27 |
| **决策** | 全部代码和硬件设计 MIT License |

## 第三部分：发布说明模板

### 版本 V1.0.0 — "First Light" (预计 2026-08)

#### 🆕 新功能
- **Web App (PWA)**: Chrome 打开即用，Canvas 手写 + 笔画录制 + Web Bluetooth 发送
- **PWA**: 可安装到桌面、离线使用已保存作品
- **激光投影盒**: ESP32 BLE 接收 → 振镜扫描 → 红绿双色墙面投影
- **AA 电池**: 4 节即装即用
- **多重安全**: 静止关断 + 晃动关断 + 物理开关

#### ⚠️ 已知问题
- iOS Safari 不支持 Web Bluetooth (请用 Chrome Android 或 iOS Bluefy)
- 仅红绿双色 (RGB V1.1)
- 室内暗光 1-3m 最佳

#### 📝 平台要求
- Android + Chrome ≥ 56
- Desktop Chrome / Edge
- 另购 4 节 AA 电池

## 附录 A: 给女朋友的卡片

> 印在包装里的文案：

**这是 InkLight，一个能把你的字变成光的小盒子。**

怎么用：
1. 装 4 节 AA 电池
2. 打开开关
3. 手机 Chrome 打开 inklight.app
4. 点"连接设备"→选 InkLight
5. 写字 → 点投影
6. 看墙上——你的字一笔一画亮起来

小提醒：
- 激光别直射眼睛哦
- 没电了换电池，不用充电
- 拍照发朋友圈！(别忘了 @做这个盒子的人 😄)
